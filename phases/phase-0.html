Система авторизации и сессий

Задачи:

  • [ ] Спланировать систему авторизации и сессий
  • [ ] Решить вопрос с refresh tokens
  • [ ] Спроектировать auth guards
  • [ ] Определить хранение токенов

Ключевые решения:

// Token storage
- Access token в memory
- Refresh token в httpOnly cookie (preferred) или localStorage

// Auth flow
1. Login → получение access + refresh tokens
2. Сохранение токенов
3. Axios interceptor для добавления access token
4. Axios interceptor для refresh при 401
5. Logout → очистка токенов

// Auth guards
- requireAuth() - для защищенных страниц
- requireGuest() - для login/register
- requireKYC() - для определенных операций
