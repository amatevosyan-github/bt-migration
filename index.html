Миграция на Vue 3 + Vite + Vuetify

Техническая документация проекта объединения трех фронтенд приложений BIT.TEAM в единое SPA

Начать изучениеФазы миграции

🎯 Maintainability

Единая кодовая база, современный стек технологий, модульная архитектура для легкой поддержки и развития

📈 Scalability

Архитектура, готовая к росту - от модульной структуры до оптимизированной сборки

🎨 Единый Design System

UI Kit на базе Vuetify с компонентами из Figma, поддержка Dark/Light тем

⚡ Современный стек

Vue 3 Composition API, Vite 5+, Vuetify 3.5+, Pinia 2, TypeScript

🔄 Постепенная миграция

Поэтапный подход от Auth модуля до полного функционала SPOT и P2P

🚀 Production Ready

CI/CD, тестирование, мониторинг, оптимизация производительности

🏗️ Текущее состояние

Три отдельных фронтенд приложения:

  • exchange-website [SPOT] - Nuxt 2
  • next-bit.team-react-front [P2P] - Next.js 14
  • bit-team-auth-front [Auth] - React 18

🎯 Целевое состояние

Единое SPA приложение на Vue 3 + Vite + Vuetify с:

  • ✨ Единой дизайн-системой из Figma
  • 🌓 Dark/Light темами
  • 📦 Внутренней UI библиотекой на базе Vuetify
  • 🔧 Модульной архитектурой
  • 🚀 Оптимизированной сборкой

📋 Этапы миграции

Миграция разделена на 12 фаз с четкими MVP этапами:

ЭтапОписаниеДлительность
MVP 1Базовая функциональность + Auth~6-7 недель
MVP 2SPOT trading полностью~4-5 недель
MVP 2.5Публичные страницы~1-2 недели
MVP 3P2P модуль~5+ недель
ProductionОптимизация и деплой~2-3 недели
