Обзор проекта

Текущее состояние

Существующие приложения

1. exchange-website [SPOT]

  • Технологии: Nuxt 2
  • Функционал: SPOT трейдинг, кошельки, рынки
  • Особенности: Server-side rendering, устаревший стек

2. next-bit.team-react-front [P2P]

  • Технологии: Next.js 14
  • Функционал: P2P торговля

3. bit-team-auth-front [Auth]

  • Технологии: React 18
  • Функционал: Авторизация, регистрация, 2FA, KYC

Проблемы текущей архитектуры

Вызовы

  • 🔄 Дублирование кода - общие компоненты и логика копируется между проектами
  • 🎨 Несогласованный UI - разные UI библиотеки (Vuetify 2, Material UI, custom)
  • 🔧 Сложность поддержки - три разных стека технологий
  • 📦 Фрагментированные зависимости - разные версии библиотек
  • 🚀 Медленная разработка - изменения требуют работы в нескольких репозиториях

Целевое состояние

Единое SPA приложение

Приложение на Vue 3 + Vite + Vuetify 3 с фокусом на:

🛠️ Maintainability (Поддерживаемость)

  • Единая кодовая база
  • Современный стек технологий
  • Модульная архитектура
  • TypeScript для type safety
  • Единый CI/CD pipeline

📈 Scalability (Масштабируемость)

  • Модульная структура проекта
  • Ленивая загрузка модулей
  • Code splitting
  • Оптимизированная сборка
  • Готовность к новым модулям

🎨 Единая дизайн-система

  • UI Kit на базе Vuetify 3
  • Компоненты из Figma
  • Atomic Design подход
  • Dark/Light темы
  • Consistent UX

Структура проекта

├── src/
│   ├── app/                    # App configuration

│   │   ├── p2p/               # P2P торговля
│   │   ├── wallet/            # Кошельки
│   │   ├── profile/           # Профиль
│   │   ├── home/              # Публичные страницы
│   │   └── shared/            # Общие модули
│   ├── shared/                # Shared resources
│   │   ├── ui/                # UI Kit (Atomic Design)
│   │   ├── components/        # Общие компоненты

├── public/                    # Public assets
└── tests/                     # Тесты

Модульная архитектура

Каждый модуль содержит:

modules/[module-name]/
├── pages/           # Страницы модуля
├── components/      # Компоненты модуля
├── composables/     # Composables модуля
├── stores/          # Pinia stores модуля
└── routes.js        # Маршруты модуля

Преимущества:

  • Изолированная логика
  • Легкое масштабирование
  • Независимое тестирование
  • Code splitting по модулям

Технические решения

State Management

Pinia 2 для управления состоянием:

  • ✅ TypeScript support из коробки
  • ✅ Composition API style
  • ✅ Модульная структура stores

API Layer

Единый API сервисный слой:

  • Axios для HTTP запросов
  • Interceptors для auth
  • Request/Response transformers
  • Error handling
  • Retry logic

WebSocket

Real-time обновления:

  • Цены и тикеры
  • Order book
  • User orders/trades
  • Уведомления

Routing

Vue Router 4 с:

  • Ленивая загрузка страниц
  • Auth guards
  • Meta information
  • Dynamic routes
  • Nested routes

Интернационализация

Vue I18n 9:

  • Английский и русский языки
  • Ленивая загрузка локалей
  • Pluralization
  • Number/Date formatting

Миграционная стратегия

Поэтапный подход

Миграция разбита на фазы с четкими MVP этапами:

  1. Фаза 0-1: Подготовка и инициализация
  2. Фаза 2-4: UI Kit и Auth модуль (MVP 1)
  3. Фаза 5: SPOT модуль (MVP 2)
  4. Фаза 6: P2P модуль (MVP 3)
  5. Фазы 7-12: Дополнительные модули и оптимизация

Принципы миграции

Подход

  • ✅ Постепенная миграция - по одному модулю
  • ✅ Переиспользование логики - адаптация существующего кода
  • ✅ Тестирование на каждом этапе - проверка функциональности
  • ✅ Демо версии - регулярные релизы для проверки
  • ✅ Параллельная разработка - старое работает, пока новое разрабатывается
